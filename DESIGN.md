# DESIGN SYSTEM - Open RaceRender Web (Carbon Core)

Ugg! Caveman make strong design for fast car dashboard! Web app must look sleek, look premium, make user go "WOW!" at first sight.

This document defines the visual language, typography, layout, and components for the Open RaceRender Web application.

---

## 1. Visual Language & Color Palette

The design is built on a dark carbon aesthetic with glowing neon accents reminiscent of high-end racing data systems.

| Token                   | CSS Class                       | Color                          | Purpose                                     |
| :---------------------- | :------------------------------ | :----------------------------- | :------------------------------------------ |
| **Dark Void (BG)**      | `bg-zinc-950`                   | `#09090b`                      | Base application background                 |
| **Carbon Glass (Card)** | `bg-zinc-900/80`                | `#18181b` (opacity 80% + blur) | Dashboard cards, control panels             |
| **Crimson Peak**        | `text-rose-500` / `bg-rose-500` | `#f43f5e`                      | Redline warnings, hot laps, alarms          |
| **Electric Cyan**       | `text-cyan-400` / `bg-cyan-400` | `#22d3ee`                      | Active vectors, path trace, sync markers    |
| **Muted Slate**         | `text-zinc-500`                 | `#71717a`                      | Unimportant metadata, secondary unit labels |
| **Bright White**        | `text-zinc-100`                 | `#f4f4f5`                      | Crucial telemetry readings, headings        |

### Effects

- **Glassmorphism**: Backdrop blur `backdrop-blur-md` on card panels.
- **Glow Elements**: `drop-shadow-[0_0_8px_rgba(34,211,238,0.4)]` for telemetry dots and active indicators.

---

## 2. Typography

- **System Font**: `Inter` or `Geist` (clean, highly readable sans-serif for settings, controls, lists).
- **Telemetry Font**: `monospace` / `Orbitron` (digital readout style, perfect for fast-changing numbers like Speed and RPM to prevent visual layout shifts).

---

## 3. Core Telemetry Overlay Gauges

All overlays must render beautifully client-side at 60fps on top of the POV video container.

### A. Speedometer (Circular Dial / Digital Gauge)

- Beautiful SVG dial representing speed up to 300 km/h or 180 mph.
- Neon blue active indicator sweeping around the rim.
- Large center digital readout with Orbitron-style typography.

### B. RPM & Shift Lights (Horizontal Bar / Sweeping Gauge)

- Linear bar showing engine speed from 0 to 10,000 RPM.
- Segmented LED blocks:
  - `0 - 6,000 RPM`: Soft Green segments
  - `6,000 - 8,500 RPM`: Warm Yellow segments
  - `8,500 - 9,500 RPM`: Alert Crimson segments (Redline)
  - `> 9,500 RPM`: Blashing Violet/Blue (Shift Warning!)

### C. G-Force Radar (Concentric Target)

- Concentric circle plot representing 0.5G, 1.0G, 1.5G limits.
- Horizontal (Lateral Accel) and Vertical (Longitudinal Accel) grid lines.
- Red glowing bubble dot representing current Gs.
- Muted red tail trailing the last 5 frames of G-force coordinates for fluid movement representation.

### D. Track Map Circuit Outline

- Autogenerated track map path drawn dynamically using latitude/longitude telemetry coordinates.
- Full track drawn in a thin muted zinc line.
- Vehicle's current position represented as a bright cyan pulsing dot moving along the track.
- Clean scale representation in bottom-left corner of track card.

## 4. Layout & Drag-and-Drop Workspace

The workspace is split into two modes:

### Layout Sync/Edit Mode

- **Left Panel (60% width)**: Big video screen with overlay layers. Widgets can be toggled on/off and customized.
- **Right Panel (40% width)**: Upload controls, sync tools, telemetry channel mapping, and project settings.
- **Bottom Panel**: Video timeline scrubber aligned with telemetry index scrubber for precise sync alignment.

### Presentation/Export Mode

- Fullscreen video layout with absolute overlays ready for real-time play or client-side screen recording capture.
